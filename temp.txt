Here is the complete Phase 1 roadmap, mapping each team member's responsibilities directly to the exact files they need to create and edit in your repository. 

### **Member 1: Data & Batching Lead**
**Focus:** Data ingestion, universal parsing rules, and batch splitting.
* **Files to Edit:**
    * `src/controller/utils.py` (or a dedicated `batch_processor.py`)
    * `.gitignore`
* **Roadmap:**
    1. Download the NASA logs and place them locally in `data/raw/`.
    2. Immediately update the `.gitignore` to ensure those massive `data/raw/` and `data/output/` files are never committed to GitHub.
    3. Finalize the master Regex parsing string that cleanly extracts all required fields. 
    4. Write the Python logic in `utils.py` that reads the raw logs and splits them into smaller, sequentially numbered batch files (e.g., `batch_1.txt`, `batch_2.txt`).

### **Member 2: MapReduce Core Developer**
**Focus:** The Java execution engine running in local Standalone mode.
* **Files to Edit:**
    * `src/pipelines/mapreduce/pom.xml`
    * `src/pipelines/mapreduce/src/main/java/com/team/etl/LogDriver.java`
    * `src/pipelines/mapreduce/src/main/java/com/team/etl/LogMapper.java`
    * `src/pipelines/mapreduce/src/main/java/com/team/etl/LogReducer.java`
* **Roadmap:**
    1. Define the Hadoop Core dependencies in `pom.xml`.
    2. Write the `LogMapper.java` utilizing Member 1's exact Regex. Implement Hadoop Counters to track malformed records and handle the missing byte (0) logic.
    3. Write the `LogReducer.java` to aggregate the data for Queries 1, 2, and 3.
    4. Configure `LogDriver.java` to accept local file paths (`file:///`) for input and output, bypassing HDFS entirely. Compile this into the final executable `.jar`.

### **Member 3: Database & Analytics Lead**
**Focus:** Relational storage, schema design, and Python ingestion.
* **Files to Edit:**
    * `database/schema.sql`
    * `src/controller/db_client.py`
* **Roadmap:**
    1. Install and configure PostgreSQL (spinning this up within a WSL/Ubuntu environment keeps the local networking and port management very clean).
    2. Write `schema.sql`. Given the need to evaluate query logic and maintain data integrity, carefully design the tables so the mandatory outputs perfectly map to the `Run_Metadata` (batch ID, pipeline, runtime).
    3. Write the Python functions in `db_client.py` using `psycopg2` to read the MapReduce output text files and execute the `INSERT` statements.

### **Member 4: Orchestrator & UI Lead**
**Focus:** The central Python controller, system integration, and final reporting.
* **Files to Edit:**
    * `src/controller/main.py`
    * `src/controller/reporting.py`
* **Roadmap:**
    1. Build the interactive terminal interface in `main.py` where the user selects the pipeline and batch size.
    2. Write the `subprocess` logic in `main.py` to trigger Member 1's batcher, then execute Member 2's Java `.jar`, and finally call Member 3's `db_client.py` insertion functions. Manage the runtime timers here.
    3. Write `reporting.py` to connect to PostgreSQL, fetch the final statistics, and print the beautifully formatted tables and execution metadata to the console.
